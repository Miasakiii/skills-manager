## 一、项目概述

### 1.1 一句话描述

一个统一管理、分类浏览、打包分发 AI Agent Skills 的本地工具，支持将 Skills 适配输出到主流 Agent 平台。

### 1.2 解决的问题

当前 AI Agent 生态中，「工具/技能」（Tool / Skill / Plugin）的管理方式高度碎片化：

- 每个 Agent 框架有自己的工具定义格式（OpenAI、Claude、Gemini、LangChain……）
- 工具散落在各个项目目录中，没有统一的存储和检索方式
- 从网上获取的工具缺乏标准化的安装、卸载、版本管理流程
- 没有统一的分类浏览和搜索机制，难以快速找到需要的能力
- 在不同 Agent 之间迁移时，工具定义需要手动改写

### 1.3 核心价值

1. **集中管理**：所有 Skills 统一存储在本地，安装、卸载、更新一条命令搞定
2. **分类浏览**：按类别、标签、关键词快速浏览和搜索已安装的 Skills
3. **多平台适配**：一份 Skill 定义，一键导出为 OpenAI / Claude / Gemini / LangChain / MCP 等格式
4. **打包分发**：通过 `.skill` 压缩包或 URL 安装 Skills，方便分享和复用

### 1.4 非目标（明确不做）

以下功能在当前阶段 **不做**，但架构上预留扩展空间：

- ~~远程 Registry / 插件市场~~（初期用 GitHub Releases 替代）
- ~~Docker/WASM 沙箱执行~~（初期信任本地执行）
- ~~局域网发现与共享~~
- ~~Dify/Coze 等平台级深度适配~~（v2 考虑）
- ~~完整的依赖解析系统~~（仅支持声明式依赖，不做自动传递解析）
- ~~GUI 桌面应用~~（v2 考虑，初期 CLI + 可选 HTML 浏览页）

## 二、核心概念定义

### 2.1 什么是 Skill

一个 **Skill** 是一个自包含的 AI 能力单元，它可以是以下任意一种或多种的组合：

| Skill 类型 | 说明 | 示例 |
|---|---|---|
| `prompt` | Prompt 模板，定义 Agent 的指令或行为模式 | 翻译 prompt、代码审查 prompt |
| `tool` | 可调用的函数/工具，接受输入返回输出 | 调用 API、执行计算、操作文件 |
| `workflow` | 多步骤编排，串联多个操作 | 搜索 → 总结 → 输出报告 |
| `knowledge` | 知识库片段，供 RAG 检索 | 领域文档、参考资料 |
| `composite` | 以上类型的组合 | 包含 prompt + tool + knowledge 的完整能力 |

### 2.2 Skill 的生命周期

创建 → 打包 → 安装 → 启用 → 绑定到 Agent → 使用 → 更新 → 卸载
↑ │
└─── 禁用 ←─────────────────┘

### 2.3 关键术语

| 术语 | 定义 |
|---|---|
| **Skill** | 一个自包含的能力单元 |
| **Skill Package** | 打包后的 Skill（`.skill` 压缩文件或目录） |
| **Store** | 本地存储目录 `~/.skills/store/`，存放所有已安装的 Skills |
| **Catalog** | 本地 Skills 的索引数据，支持分类浏览和搜索 |
| **Adapter** | 格式适配器，将 Skill IR 转换为目标平台的工具定义格式 |
| **IR (Intermediate Representation)** | Skill 的统一中间表示，所有适配器的输入 |
| **Profile** | Agent 的 Skill 配置文件，记录该 Agent 使用了哪些 Skills |

### 3.2 skill.yaml 完整字段定义

```yaml
# ============================================================
# 必需字段
# ============================================================
name: "smart-translator"          # Skill 唯一标识（小写 + 连字符，npm 包名风格）
version: "1.2.0"                  # 语义化版本号
type: "composite"                 # 类型：prompt | tool | workflow | knowledge | composite
display_name: "智能翻译"           # 人类可读的名称
description: >                    # 一句话描述（不超过 100 字）
  支持上下文感知的多语言翻译，
  自动检测源语言并保持专业术语一致性。

# ============================================================
# 分类与标签
# ============================================================
category:
  primary: "language"             # 一级分类（见分类体系）
  secondary: ["translation"]     # 二级标签

tags: ["translation", "i18n", "multilingual", "nlp"]  # 自由标签
domain: ["general", "business"]  # 适用领域

# ============================================================
# 能力摘要（给人看的，快速理解用）
# ============================================================
capabilities_summary:
  - "翻译文本到 7 种语言（中/英/日/韩/法/德/西）"
  - "自动检测源语言"
  - "支持 formal / casual / technical 三种风格"
  - "返回翻译置信度"

# ============================================================
# 适用 / 不适用场景
# ============================================================
use_cases:
  - "翻译用户反馈或评论"
  - "多语言内容本地化"
  - "跨语言文档对比"

not_for:
  - "实时口语翻译（延迟较高）"
  - "文学作品翻译（缺乏文学润色能力）"

# ============================================================
# 能力定义（tool 类型的关键部分）
# ============================================================
capabilities:
  - name: "translate"
    description: "将文本翻译到目标语言"
    input:
      type: object
      properties:
        text:
          type: string
          description: "待翻译的文本"
          minLength: 1
          maxLength: 10000
        target_lang:
          type: string
          description: "目标语言代码"
          enum: [zh, en, ja, ko, fr, de, es]
        style:
          type: string
          description: "翻译风格"
          enum: [formal, casual, technical]
          default: formal
      required: [text, target_lang]
    output:
      type: object
      properties:
        translated_text:
          type: string
        confidence:
          type: number
        detected_source_lang:
          type: string
    execution:
      mode: sync                  # sync | async | streaming
      timeout: 30000              # 超时（毫秒）
      idempotent: true            # 是否幂等

  - name: "batch_translate"
    description: "批量翻译多个文本"
    # ... 类似结构

# ============================================================
# 执行配置
# ============================================================
executor:
  type: "python"                  # python | node | shell | http
  entry: "tools/handler.py"       # 入口文件
  function: "translate"           # 入口函数名（Python/Node 时）
  requirements:                   # 运行时依赖
    - "requests>=2.28"
    - "langdetect>=1.0"

# ============================================================
# 配置项声明（用户安装后需要填写的）
# ============================================================
config:
  api_key:
    type: string
    secret: true
    description: "翻译服务 API Key"
    required: true
  api_provider:
    type: string
    enum: [google, deepl, baidu]
    default: google
    description: "翻译 API 提供商"
  default_style:
    type: string
    enum: [formal, casual, technical]
    default: formal
    description: "默认翻译风格"

# ============================================================
# 依赖（其他 Skills）
# ============================================================
dependencies:
  - name: "language-detector"
    version: ">=1.0.0"
    optional: false

# ============================================================
# 错误码声明
# ============================================================
errors:
  - code: "LANGUAGE_NOT_SUPPORTED"
    description: "不支持的目标语言"
  - code: "TEXT_TOO_LONG"
    description: "文本超出长度限制"
  - code: "API_ERROR"
    description: "翻译 API 调用失败"

# ============================================================
# 安全声明
# ============================================================
security:
  requires_network: true
  requires_api_key: true
  data_sends_to: "第三方翻译 API（Google/DeepL/百度）"
  file_access: "none"

# ============================================================
# 作者信息
# ============================================================
author: "someone"
license: "MIT"
homepage: "https://github.com/someone/smart-translator"
repository: "https://github.com/someone/smart-translator"
min_skills_version: "0.1.0"      # 要求的 Skills Manager 最低版本

3.3 打包格式

Skill 包使用 .skill 扩展名，实质为 .tar.gz：
smart-translator-1.2.0.skill
  └── smart-translator/          # 顶层目录 = skill name
      ├── skill.yaml
      ├── README.md
      ├── prompts/
      ├── tools/
      └── ...
打包命令：
skills pack ./my-skill/          # → my-skill-1.0.0.skill
也支持 .zip 格式的自动识别和安装。

五、功能规格

5.1 Skill 管理

5.1.1 安装 Skill

从本地目录安装：
skills install ./my-skill/
读取 skill.yaml，校验格式
复制到 ~/.skills/store/my-skill/
更新 SQLite 索引
如果缺少分类/描述信息，提示是否自动补全

从压缩包安装：
skills install my-skill-1.0.0.skill
skills install my-skill.zip
解压到临时目录
校验 skill.yaml 存在且合法
复制到 store
清理临时目录

从 URL 安装：
skills install https://github.com/user/repo/releases/download/v1.0/my-skill-1.0.0.skill
skills install https://example.com/my-skill.zip
下载到 ~/.skills/cache/
按压缩包流程安装

从 GitHub 仓库安装：
skills install github:user/repo
skills install github:user/repo@v1.2.0      # 指定 tag
下载仓库的指定 tag/branch 的 zip
在仓库根目录或指定目录查找 skill.yaml
按目录流程安装
安装选项：
skills install ./my-skill/ --name my-alias   # 自定义安装名
skills install ./my-skill/ --force            # 覆盖已有安装
skills install ./my-skill/ --no-enrich        # 跳过自动元数据补全

5.1.2 卸载 Skill
skills uninstall smart-translator
skills uninstall smart-translator --purge     # 同时删除用户配置
从 store 删除目录
从 SQLite 索引删除记录
检查是否有其他 Skill 依赖此 Skill，给出警告

5.1.3 更新 Skill
skills update smart-translator                # 从原来源重新安装最新版
skills update --all                           # 更新所有可更新的 Skills

5.2.4 自动元数据补全

当安装的 Skill 缺少分类或描述信息时：

skills enrich smart-translator
skills enrich --all                  # 补全所有缺少信息的 Skills
工作流程：


1.读取 Skill 的所有文件内容（skill.yaml、README、代码、prompt 等）
2.调用 LLM（通过用户配置的 API）生成：
一句话中文描述
一级分类建议
二级标签建议
能力要点列表
适用/不适用场景
3.展示给用户确认
4.写入本地元数据（不修改原始 skill.yaml，而是存在 catalog.db 中）

LLM 调用配置（在 ~/.skills/config.yaml 中）：

enrichment:
  enabled: true
  provider: "openai"              # openai | claude | local
  model: "gpt-4o-mini"
  api_key: "sk-..."
  base_url: "https://api.openai.com/v1"   # 兼容接口地址
  language: "zh"                  # 生成语言

七、SQLite 索引数据库设计

表结构
-- Skills 主表
CREATE TABLE skills (
    id              INTEGER PRIMARY KEY AUTOINCREMENT,
    name            TEXT UNIQUE NOT NULL,
    display_name    TEXT,
    version         TEXT NOT NULL,
    type            TEXT NOT NULL,           -- prompt/tool/workflow/knowledge/composite
    description     TEXT,
    author          TEXT,
    license         TEXT,
    homepage        TEXT,
    installed_at    DATETIME DEFAULT CURRENT_TIMESTAMP,
    updated_at      DATETIME DEFAULT CURRENT_TIMESTAMP,
    source          TEXT,                    -- 安装来源（路径/URL/GitHub）
    source_ref      TEXT,                    -- 来源引用（tag/commit）
    enabled         BOOLEAN DEFAULT 1,
    enriched        BOOLEAN DEFAULT 0        -- 是否已补全元数据
);

-- 分类表
CREATE TABLE categories (
    id              INTEGER PRIMARY KEY AUTOINCREMENT,
    skill_id        INTEGER NOT NULL,
    primary_cat     TEXT NOT NULL,            -- 一级分类
    secondary       TEXT,                     -- 二级标签，JSON 数组
    domain          TEXT,                     -- 适用领域，JSON 数组
    FOREIGN KEY (skill_id) REFERENCES skills(id) ON DELETE CASCADE
);

-- 标签表（用于搜索）
CREATE TABLE tags (
    id              INTEGER PRIMARY KEY AUTOINCREMENT,
    skill_id        INTEGER NOT NULL,
    tag             TEXT NOT NULL,
    FOREIGN KEY (skill_id) REFERENCES skills(id) ON DELETE CASCADE
);
CREATE INDEX idx_tags_tag ON tags(tag);

-- 能力摘要表
CREATE TABLE capability_summaries (
    id              INTEGER PRIMARY KEY AUTOINCREMENT,
    skill_id        INTEGER NOT NULL,
    summary         TEXT NOT NULL,
    sort_order      INTEGER DEFAULT 0,
    FOREIGN KEY (skill_id) REFERENCES skills(id) ON DELETE CASCADE
);

-- 场景表
CREATE TABLE scenarios (
    id              INTEGER PRIMARY KEY AUTOINCREMENT,
    skill_id        INTEGER NOT NULL,
    type            TEXT NOT NULL,            -- 'use_case' 或 'not_for'
    description     TEXT NOT NULL,
    FOREIGN KEY (skill_id) REFERENCES skills(id) ON DELETE CASCADE
);

-- Agent Profiles 表
CREATE TABLE profiles (
    id              INTEGER PRIMARY KEY AUTOINCREMENT,
    name            TEXT UNIQUE NOT NULL,
    description     TEXT,
    created_at      DATETIME DEFAULT CURRENT_TIMESTAMP
);

-- Profile-Skill 关联表
CREATE TABLE profile_skills (
    profile_id      INTEGER NOT NULL,
    skill_id        INTEGER NOT NULL,
    sort_order      INTEGER DEFAULT 0,
    PRIMARY KEY (profile_id, skill_id),
    FOREIGN KEY (profile_id) REFERENCES profiles(id) ON DELETE CASCADE,
    FOREIGN KEY (skill_id) REFERENCES skills(id) ON DELETE CASCADE
);

-- 全文搜索（FTS5）
CREATE VIRTUAL TABLE skills_fts USING fts5(
    name, display_name, description, tags_text,
    content='skills',
    content_rowid='id'
);

tools/handler.py：
import json


def format_json(text: str, indent: int = 2, mode: str = "beautify") -> dict:
    try:
        data = json.loads(text)
        if mode == "minify":
            result = json.dumps(data, ensure_ascii=False, separators=(",", ":"))
        else:
            result = json.dumps(data, ensure_ascii=False, indent=indent)
        return {"result": result, "valid": True, "error": None}
    except json.JSONDecodeError as e:
        return {"result": None, "valid": False, "error": str(e)}


def validate_json(text: str) -> dict:
    try:
        json.loads(text)
        return {"valid": True, "error": None}
    except json.JSONDecodeError as e:
        return {"valid": False, "error": str(e)}

十、风险与对策

风险	影响	对策
各平台 API 频繁变动	适配器需要持续维护	适配器设计为独立模块，便于单独更新；建立平台 API 变动监控
Skill 格式规范过于复杂	用户不愿按规范写	提供 skills init --template 快速生成骨架；支持最小化字段（只需 name + version + type）
用户不知道怎么写 Skill	生态起不来	提供详细的示例 Skills；提供 Skill 模板生成器
LLM 自动补全不准确	分类和描述质量差	用户可手动覆盖；支持多轮修正；将补全结果作为建议而非强制
与现有工具（npm/pip）定位重叠	用户困惑	明确定位：这是 AI Agent 层面的 Skill 管理，不是通用包管理器
